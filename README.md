# mktopic-notes

## 项目简介

`mktopic-notes` 是一个用于归档和发布多个独立 HTML 页面的仓库。本项目不针对任何特定域名或网站，仅作为通用页面存档与展示用途。

适合存放各类静态内容、笔记页面、临时演示页面等，便于通过 GitHub Pages 或其他静态托管服务访问。

## 目录结构说明

```
mktopic-notes/
├── index.html            # 仓库首页（可选）
├── pages/                # 存放主要 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/               # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── README.md             # 本文件
└── .gitignore
```

- `pages/`：建议将每个独立页面放置于此目录下，便于管理。
- `assets/`：存放页面共用的样式、脚本或图片资源。如果页面资源独立，也可在其所在目录下自建子目录。

## 页面归档说明

- 本仓库中每个 HTML 文件均为独立的归档页面，彼此之间无强制关联。
- 页面内容以笔记、资料整理或演示为主，不包含商业推广、落地页或跳转链接。
- 文件名建议采用有意义的英文或拼音命名，避免特殊字符。

## 维护说明

- 欢迎提交 Pull Request 新增或更新页面，但请确保内容符合仓库定位。
- 不建议在此仓库中放置大量外部依赖或动态脚本，保持页面轻量与可离线查看。
- 如需启用 GitHub Pages，可在仓库设置中开启，并选择 `main` 分支作为源。
